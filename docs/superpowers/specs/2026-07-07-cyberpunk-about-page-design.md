# 炫酷关于页面设计（赛博朋克风格）

- 日期：2026-07-07
- 状态：待用户审查

## 背景

站点是基于 Chirpy 主题（`jekyll-theme-chirpy` 7.5.0）的个人技术博客，暗色模式（`theme_mode: dark`），站点名 "Mrhan Blog"，标语 "系统恒久远, 代码永流传"，头像 `assets/img/rick.png`。`_tabs/about.md` 目前仍是主题默认占位文案，从未定制过。仓库里存在一个未使用的 `assets/styles/about.css` 空样式文件，可以直接利用起来。

目标：把关于页面改造成一个具有赛博朋克科技感的、有炫酷交互效果的个人主页，而不是纯文字页面。

## 视觉风格

赛博朋克科技感（Cyberpunk）：深色背景 + 霓虹紫 / 青 / 绿色系点缀，科技感边框与发光效果，配合等宽字体（终端风格）用于关键文案。

## 交互效果

页面同时包含以下四种效果，效果触发节奏带随机性（例如 Glitch 抖动的触发间隔随机、打字机文案随机从多条备选中选择一条播放），而不是四种效果轮流出现或整页刷新切换：

1. **打字机效果**：Hero 区文案逐字输出，光标闪烁
2. **粒子背景动画**：Canvas 2D 实现的漂浮粒子 + 邻近连线，作为全视口固定背景
3. **文字 Glitch 效果**：姓名/标题类文字持续有轻微的错位、色差跳动
4. **霓虹发光边框**：技能标签、联系图标的边框呈现呼吸式发光，hover 时增强

## 内容结构

自上而下：

1. **Hero 区**
   - 头像（复用现有 `rick.png`）
   - 打字机效果播放的一行 slogan（占位程序员人设文案，多条备选随机挑一条播放）
   - 姓名/标题，应用 Glitch 效果
2. **技能栈展示区**
   - 一组霓虹发光边框标签，占位技能名称。默认标签取自现有 `_posts` 目录下的分类（C++、Java、OS、Python、Rust、前端、容器、数据库、构建工具、AI 等），后续由用户自行替换为准确技能
3. **底部联系区**
   - 一行发光图标链接：GitHub（`https://github.com/mrhanhan`）、邮箱（读取 `_config.yml` 中 `social.email` 当前值）、语雀、Gitee（与现有 `_panels/` 面板保持一致）
   - hover 时霓虹发光增强

不包含时间线/经历模块（本次范围排除）。

## 技术方案

- **改动范围严格限制**：只改 `_tabs/about.md`（内嵌内容片段，kramdown 支持行内 HTML）+ 新增 `assets/js/about-fx.js` + 完善 `assets/styles/about.css`。不改动 `_layouts/panel.html` 等全局模板，不影响其他页面/侧边栏。
- **布局策略**：粒子背景用 `position: fixed; z-index: -1;` 铺满整个视口，不参与文档流、不影响 Chirpy 现有栅格布局；Hero/技能/联系方式等前景内容仍在 Chirpy 原有的内容列宽度内，以"深色玻璃卡片"容器呈现，不做突破布局宽度限制的处理（避免和主题栅格系统产生冲突、避免升级主题后被破坏）。
- **粒子背景**：原生 Canvas 2D 手写实现，不引入第三方库（避免额外网络请求、避免主题升级/依赖冲突风险）。
- **不特别处理浅色模式**：当前站点固定 `theme_mode: dark`；若用户以后手动切换为浅色模式，霓虹配色可能不协调，属于已知的可接受限制，本次不做浅色模式适配。

## 兼容性与降级

- `prefers-reduced-motion: reduce`：关闭粒子背景动画、Glitch 抖动、霓虹呼吸动画，改为静态样式；打字机效果直接展示完整文本（不做逐字动画）
- 无 JS 环境：`<noscript>` 兜底，Hero 区退化为纯静态文本 + 头像，页面基本内容仍可读
- 移动端（窄屏）：粒子数量减半（性能考虑），发光效果保留但简化阴影层数，技能标签自动换行，底部联系图标行不溢出

## 验证方式

- 本地 `bundle exec jekyll serve` 起服务，浏览器实际验证暗色模式下打字机 / 粒子 / Glitch / 霓虹效果是否正常运行
- 窄屏（375px）下检查技能标签换行、联系图标行是否溢出
- 用 Lighthouse 快速检查性能与无障碍是否有严重问题（纯 CSS/Canvas 实现，理论上无额外网络请求开销）
- 验证 `prefers-reduced-motion` 开启时效果是否正确降级为静态

## 范围之外（本次不做）

- 时间线/经历模块
- 浅色模式下的霓虹配色适配
- 引入第三方粒子库或动画库
- 突破 Chirpy 内容列宽度限制的全屏沉浸式布局
